getStores لا يُطبق type filter أبداً
السطر: const { type } = req.query موجود لكن لا يوجد .eq('type', type) في الـ query. الفلتر في صفحة الهوم الرئيسية لا يعمل. يجب أيضاً تصفية المتاجر التي admin_disabled=true.
modules/stores/storeController.js → getStores

!
إيراد المرتشنت يشمل delivery_fee بالخطأ
getMerchantStats يجمع o.total_price الذي يساوي subtotal + delivery_fee. الـ $3 delivery fee تذهب للسائق لا للمرتشنت. يجب: acc + (Number(o.subtotal) || Number(o.total_price) - 3).
modules/orders/orderController.js → getMerchantStats

!
earnings السائق محسوبة بثابت لا من DB
totalEarnings = deliveriesCount × DELIVERY_FEE(3.00) hardcoded. لو تغيرت الرسوم مستقبلاً، التاريخ كله خاطئ. الحل: SELECT SUM(delivery_fee) FROM orders WHERE driver_id = X AND status = 'completed'.
modules/delivery/deliveryController.js → getDriverStats
حساب
!
Race condition في addToCart يتجاوز one-store rule
الـ check ثم insert ليسا atomic. طلبان متزامنان من متجرين مختلفين قد يمران معاً. الحل: Supabase RPC function تفعل check + insert في transaction واحدة، أو إضافة UNIQUE constraint + error handling.
modules/cart/cartController.js → addToCart

ما تفعله أنت في Supabase Dashboard
1
تفعيل Row Level Security على جميع الجداول
اذهب إلى Supabase Dashboard → Table Editor → كل جدول → Enable RLS. ثم أضف policies: مثلاً orders يقرأها صاحبها فقط. هذا يحميك حتى لو backend أخطأ. بدونه أي شخص يعرف Supabase URL يقرأ كل البيانات.
Supabase Dashboard
2
إنشاء Storage bucket لصور المنتجات
Supabase Dashboard → Storage → New bucket → اسمه "products" → Public bucket. ثم اضبط policy تسمح للـ merchant برفع صور لمتجره فقط. هذا يُمكّن image upload الحقيقي.
Supabase Dashboard
3
تشغيل migration SQL للجداول الناقصة
Supabase → SQL Editor → شغّل ملفات migrations الموجودة: phase1_fixes.sql، phase2_migrations.sql، create_order_rpc.sql، promo_schema_fix.sql بهذا الترتيب. تحقق أن driver_earnings_log أو delivery_fee column موجود في orders.
Supabase SQL Editor
4
التسجيل في Paymob والحصول على API keys
اذهب لـ accept.paymob.com → سجّل حساب merchant → احصل على: API Key، Integration ID للبطاقات، HMAC Secret للـ webhook. ضعها في backend .env: PAYMOB_API_KEY, PAYMOB_INTEGRATION_ID, PAYMOB_HMAC.
paymob.com
5
إعداد Resend لإيميلات المشروع
سجّل في resend.com → أضف domain أو استخدم البريد الافتراضي للتطوير → احصل على API Key → ضعه في backend .env: RESEND_API_KEY. يُستخدم لـ: تأكيد التسجيل، تفاصيل الطلب، payout notifications.
resend.com
6
إعداد Google Maps API key
console.cloud.google.com → Enable: Maps JavaScript API, Geocoding API, Directions API, Places API → أنشئ API key مع HTTP referrer restriction لنطاقك. ضعه في frontend .env: NEXT_PUBLIC_GOOGLE_MAPS_KEY. يستبدل OSRM للتوجيه والـ geocoding.
Google Cloud Console
7
إعداد Firebase لـ Push Notifications
console.firebase.google.com → New project → Enable Cloud Messaging → احصل على: VAPID Key لـ Web Push، Server Key للـ backend. ضع VAPID في NEXT_PUBLIC_FIREBASE_VAPID_KEY وفي frontend وـ SERVER_KEY في backend .env.
Firebase Console
8
نشر الـ backend على Railway أو Render
nixpacks.toml موجود وجاهز للنشر. railway.app أو render.com → Connect GitHub → اختر backend folder → أضف Environment Variables من .env.example. تأكد من: PORT, JWT_SECRET, SUPABASE_URL, SUPABASE_KEY, FRONTEND_URL.
Railway / Render
9
نشر الـ frontend على Vercel
vercel.com → Import from GitHub → اختر frontend folder → أضف: NEXT_PUBLIC_API_URL (backend URL)، NEXT_PUBLIC_GOOGLE_MAPS_KEY، NEXT_PUBLIC_FIREBASE_VAPID_KEY. nixpacks.toml الخاص بالـ frontend موجود. Domain مجاني على vercel.app.
Vercel
